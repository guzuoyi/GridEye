# GridEye - 网格化时空聚合高速异常事件检测

基于网格化时空聚合与大小模型协作的高速公路异常事件全生命周期检测方法。

## 核心架构

```
视频流(25fps) → 抽帧(5fps) → YOLOv26s + ByteTrack
                                    ↓
                       网格引擎 SpatialTemporalGrid
                        (50×50px 高斯加权累加)
                                    ↓
                    空间聚类 → [count ≥ 120] → Qwen3.5 9B 双帧确认
                                    ↓
                    异常? → 锁定(50帧) → [count < 30] → Qwen 消警确认
```

## 运行

```bash
python run_v7.py
```

### 依赖

- ultralytics（YOLOv26s）
- torch
- opencv-python
- openai（LM Studio 客户端）

### 配置

编辑 `config/default.yaml`：

| 参数 | 默认 | 说明 |
|:--|:--|:--|
| CELL_SIZE | 50 | 网格边长（像素） |
| HOLD | 120 | 报警阈值（帧计数） |
| CLEAR | 30 | 消警阈值（区域最大计数） |
| MAX_COUNT | 400 | 网格计数上限 |
| GAUSS_3x3 | [[1,2,1],[2,4,2],[1,2,1]] | 高斯加权核 |
| GRADIENT_DECAY | 1/2/4/10 | 分阶段衰减（0-10s/10-15s/15-20s/20s+） |
| LOCK_FRAMES | 50 | Qwen 确认异常后锁定帧数 |

### 亮度自适应

| 亮度均值 | 检测置信度 |
|:--|:--:|
| < 40（极暗） | 0.08 |
| 40~80（昏暗） | 0.10 |
| ≥ 80（明亮） | 0.30 |

## 实验结果

| 视频 | 帧数 | 事件 | FSM(ByteTrack) | +空间冷却 | GridEye(本文) |
|:--|:--:|:--:|:--:|:--:|:--:|
| test.mp4 | 3001 | 无事件 | 0 | 0 | **0** |
| test1.mp4 | 1938 | 夜间违停 | 42 | 25 | **4** |
| test2.mp4 | 757 | 白天违停 | 143 | 86 | **1** |

GridEye 对无事件视频零误报，有事件视频 Qwen 调用较传统 FSM 减少 **90.5%** 和 **99.3%**。
